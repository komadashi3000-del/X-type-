# X-TYPE 診断テスト

日本語UIのシングルページ診断アプリです。100問の6択回答から、X-TYPEコード（例: `A-SISE`）を判定します。

## 使い方（ローカル）

1. このフォルダの `index.html` をブラウザで開きます。
2. 「診断を開始する」を押して、各セクションの設問に回答します。
3. 各セクションは全問回答で次に進めます。
4. 最終セクション完了後、診断結果が表示されます。

## ファイル構成

- `index.html`: 画面構造（メインメニュー/開始/設問/結果/詳細説明）
- `styles.css`: フルティガーエアロ調配色とレスポンシブUI
- `app.js`: 質問データ・進行管理・採点ロジック・結果生成・解説表示
- `README.md`: 使い方と採点仕様

## 採点ルール（要点）

### 回答スコア

- はい = `2`
- ややはい = `1`
- どちらも = `0.5`
- どちらでもない = `0`
- ややいいえ = `-1`
- いいえ = `-2`

### 軸判定

- **AFN（15問）**: `A/F/N` の合計点を比較し、最大の文字を採用
- **D/S（20問）**: `D` と `S` の合計点を比較し、高い方を採用
- **O/I（20問）**: `O` と `I` の合計点を比較し、高い方を採用
- **R/S（20問）**: `R` と `St` の合計点を比較し、高い方を採用（表示コードは `S`）
- **A/E（25問）**: 設問数が異なるため平均点で比較（A=12問、E=13問）

### コード生成

- ベースコード: `1文字目-2文字目3文字目4文字目`（例: `A-SIS`）
- フルコード: ベースコード + 末尾 `A/E`（例: `A-SISE`）
- 正式名称: ベースコードを24種マップに照合

## 結果ページの表示内容

- 結果は「前の結果 / 次の結果」でめくるように表示（7ページ）
- あなたのX-TYPEコード（フル/ベース/正式名称）
- 性格特性スコア（画像イメージに寄せたバー表示 + 数値内訳）
- あなたのタイプの詳しい解説
- 分類軸の解説（折りたたみ）
- 24タイプ一覧（展開表示、現在タイプを強調）
- X-TYPEの深い読み方について（敬語文）
- 要約文

## メインメニュー（追加）

- 診断本編
- 履歴
- 各タイプの詳細説明（今回実装・群選択対応）
- 各タイプ相性（準備中）
- MBTIとの違い・補完性（準備中）
- アカウント登録（準備中）

※ 先に「各タイプの詳細説明」を拡張しやすい形で実装しています。

### 各タイプの詳細説明ページ仕様

- 「各タイプの詳細説明」を押すと、診断本編とは別画面として表示されます。
- まず `A群 / F群 / N群` を選びます。
- 次に群内のベースコード（例: `A-DOR`）を選ぶと、右側に詳細本文を表示します。
- 今回の更新で `A群 / F群 / N群` の詳細本文を反映しました（群→タイプ選択で表示）。

## 解説データの管理場所（app.js）

- `axisDescriptions`
  - AFN, D/S, O/I, R/S, A/E の軸解説データ
  - 見出し、本文、強み/弱みを管理
- `typeDescriptions`
  - 24タイプの詳細データ
  - `short`, `core`, `strengths`, `weaknesses`, `relations`, `roles`, `depth` を管理
  - キーは必ずベースコード（例: `A-SIS`）
- `deepReadingSections`
  - 「X-TYPEの深い読み方」の各セクションを配列で管理

### 編集方法

- **タイプ文言の修正**: `typeDescriptions["A-SIS"]` の各項目を編集
- **軸解説の修正**: `axisDescriptions.AFN.items` などを編集
- **深い読み方の修正**: `deepReadingSections` の `title` / `body` を編集


## 回答者情報と履歴管理

- 診断開始前に以下を回答者情報として入力します。
  - 名前（ニックネーム可）
  - 生年月日（必須 / 1900〜2026）
  - 性別（男性・女性・自認回答）
- 回答履歴・結果履歴は、上記の回答者情報で紐づけて保存します。
- 履歴は結果ページの「あなたの回答・結果履歴」で、その回答者本人の情報に一致する履歴のみ表示されます。
- 7ページ目（あなたの回答・結果履歴）右下の「開発者画面」ボタンから、パスワード入力後に全回答データ（100問回答JSON）を表示できます。
  - 既定パスコードは `seiju_erosugidaro`（`app.js` の `CREATOR_PASSCODE`）です。

## 途中保存

- 回答データは `localStorage` に自動保存されます。
- ページを再読み込みしても途中から再開可能です。
- 結果画面の「回答データを初期化する」で保存データを削除できます。

## 拡張しやすい構造

- 質問文は `questionTexts` 配列で一元管理
- 各質問の採点ラベルは `labels` 配列で一元管理
- 判定処理は `computeScores` / `determineType` に分離
- 正式名称は `officialNameMap` に集約
- 解説は `axisDescriptions` / `typeDescriptions` / `deepReadingSections` に集約
